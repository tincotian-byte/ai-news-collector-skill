# AI资讯收集器 (AI News Collector)

## 功能概述

自动收集、整理、展示AI行业最新动态，形成结构化的新闻简报系统。

## 工作流程

### 1. 确定存储位置与采集范围

- 优先使用用户选定的工作文件夹，若无则使用 `{workspace}/ai-news/`
- 检查 `news/` 下是否有历史MD文件，读取最近一次采集日期
- **采集范围**：默认采集最近一周的AI新闻。若上次采集未超一周，仅采集上次采集日期之后的新内容，避免重复

### 2. 加载信源配置

读取工作目录下的 `sources.json` 信源配置文件。若不存在，自动从 Skill 目录复制默认版本。
该文件由用户自行维护，可增删信源，格式见"信源配置"章节。

### 3. 搜索AI资讯

**采集优先级：官方一手来源 > 权威科技媒体 > 其他**

**采集方式**：
1. 遍历 `sources.json` 中启用的信源，使用 `site:` + 时间范围组合搜索
2. 对于官方来源（tier=1），使用 WebFetch 直接访问其博客/新闻页，获取最新内容
3. 对于媒体来源（tier=2），使用 WebSearch 搜索关键词获取报道

**关键区分：事件时间 vs 文章时间**：
搜索引擎返回的是"文章发布时间"，但同一事件可能被反复报道。必须区分：
- `time`（事件时间）：事件实际发生的日期，如产品发布日、论文发表日。这是在 HTML 中用于时间筛选的字段
- `date`（采集日期）：Skill 执行采集的日期，用于标识采集批次

**如何判定事件时间**：
1. 优先从官方来源获取（官方博客的发布日期最可靠）
2. 若从媒体获取，需要从文章正文中提取事件实际发生时间（如"XX于3月15日发布..."），而非文章本身的发表时间
3. 若无法判定，使用 WebSearch 交叉验证，搜索"XX 发布 日期"
4. 绝对禁止将采集日期直接填入 `time` 字段

### 5. 资讯去重与处理

**去重策略（关键）**：
1. 同一事件多个信源报道 → 仅保留最权威的一条（官方 > 权威媒体 > 其他）
2. 与历史已采集资讯对比 → 读取 `news/` 下最近7天的MD文件，提取已有标题做相似度匹配
3. 标题高度相似（核心事件相同）→ 合并，不重复收录
4. **旧事新文**判定 → 如果一篇新文章报道的是几周/几月前的旧事件（如回顾、盘点），不收录为新资讯

**翻译**：英文标题和摘要翻译为中文，保留原文链接
**分类**：按厂商和技术领域打标签
**优先级**：高（重大发布/突破）、中（常规更新）、低（传闻/次要）

### 6. 生成Markdown简报

文件命名 `YYYY-MM-DD.md`，存入 `news/` 目录。格式见下方模板。

### 7. 生成/更新HTML阅读器

**关键**：浏览器 `file://` 协议下无法 `fetch()` 本地文件，必须将数据直接嵌入HTML。

**步骤**：
1. 读取 `news/` 下所有历史MD文件，解析为结构化数据
2. 基于所有资讯数据生成趋势总结
3. 将 `NEWS_DATA`、`TRENDS_DATA`、`LAST_UPDATED` 嵌入HTML的 `<script>` 区域
4. 每次采集后重新生成整个 `index.html`

**数据结构**：
```javascript
const NEWS_DATA = [
    {
        date: "2026-03-23",       // 采集日期（Skill执行当天）
        title: "资讯标题",
        company: "厂商名",
        field: "技术领域",
        priority: "high/medium/low",
        time: "2026-03-20",       // 事件时间（事件实际发生日期，非文章发表日期）
        source: "来源名称",
        sourceUrl: "来源URL（优先官方链接）",
        summary: "中文摘要",
        points: ["要点1", "要点2"]
    }
];

// 趋势按采集日期索引，HTML根据用户选择的时间范围自动聚合
const TRENDS_DATA = {
    "2026-03-23": [
        "<strong>关键趋势</strong> — 详细说明，<span class='trend-highlight'>重点词</span>会高亮",
        // 每次采集生成不超过4条趋势
    ]
};
```

**HTML功能**：
- 顶部时间Tab：今天 / 最近一周(默认) / 最近一个月 / 最近一年 / 自定义日期
- 趋势区：根据时间范围聚合对应采集批次的趋势，最多显示4条，支持高亮
- 侧栏筛选：厂商、技术领域、优先级（无日期），选中即时筛选，计数动态更新（分面搜索）
- 资讯列表：卡片式，支持完整/紧凑视图切换

## 信源配置

采集的信息来源由工作目录下的 `sources.json` 文件管理。Skill首次运行时自动从 `{skill_dir}/sources.json` 复制默认版本到工作目录。

**用户可以随时手动编辑此文件**，增加、删除、启用或禁用信源。

### 信源优先级（tier）

| tier | 含义 | 采集方式 | 可信度 |
|------|------|---------|--------|
| 1 | 官方一手来源 | 优先 WebFetch 直接访问 | 最高，事件时间以此为准 |
| 2 | 权威科技媒体 | WebSearch 关键词搜索 | 高，但需核实事件时间 |
| 3 | 研究机构 | WebSearch 或 WebFetch | 高 |

### sources.json 格式

```json
{
  "sources": [
    {
      "name": "OpenAI",
      "category": "国际AI大厂",
      "tier": 1,
      "enabled": true,
      "urls": ["openai.com/blog"],
      "keywords": ["OpenAI", "GPT", "ChatGPT"],
      "note": "GPT系列、ChatGPT更新"
    }
  ]
}
```

字段说明：
- `name`: 厂商/媒体名称
- `category`: 分类（国际AI大厂 / 国内AI大厂 / 权威科技媒体 / 研究机构 / 用户自定义）
- `tier`: 信源优先级（1=官方, 2=媒体, 3=研究机构）
- `enabled`: 是否启用（false则跳过）
- `urls`: 搜索时用 `site:` 限定的域名
- `keywords`: 搜索时使用的关键词
- `note`: 备注

> **用户增加来源**：直接编辑 `sources.json`，在 `sources` 数组中添加一条即可。也可以告知AI"增加XX为资讯来源"，Skill会自动更新该文件。

## 采集时间逻辑

```
本次采集时间: T_now
上次采集时间: T_last (从 news/ 目录最新文件日期获取)

如果 T_now - T_last < 7天:
    搜索范围 = T_last 到 T_now（仅采集增量）
否则:
    搜索范围 = T_now - 7天 到 T_now（完整一周）

去重范围 = news/ 目录下最近7天的所有MD文件
```
